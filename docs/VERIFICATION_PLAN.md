# 潜在风险验证与处理计划

## 一、问题识别总结

通过代码审查，发现以下潜在风险：

### 1. 状态管理逻辑不一致
| 模块 | 创建时状态处理 | 更新时状态处理 | 问题 |
|------|---------------|---------------|------|
| 管理员用户 (SysUser) | 总是设置为 `STATUS_ENABLED` (1)，忽略前端传入 | 可以修改状态 | 创建页面显示状态选择但无效 |
| 前端用户 (UsrUser) | 可以传入状态，默认为 `USER_STATUS_NORMAL` (1) | 可以修改状态 | 无问题 |

### 2. 密码策略差异
| 模块 | 默认密码 | 复杂度验证 | 需改密标记 |
|------|---------|-----------|-----------|
| 管理员用户 | 123456 | 无 | 无 |
| 前端用户 | 123456 | 有（123456跳过） | 有 (`needChangePassword=1`) |

### 3. 超级管理员保护机制
- 后端已实现保护逻辑
- 前端错误处理简单，没有专门的超级管理员保护提示

### 4. 前端UI一致性
- 管理员创建页面显示状态选择（正常/禁用），但后端不接受
- 前端用户创建页面没有状态选择，但后端接受

## 二、验证测试方案

### 2.1 状态管理逻辑验证

#### 测试1: 管理员创建时状态字段处理
**目标**: 验证管理员创建时发送的 `status` 字段是否被忽略

**测试步骤**:
1. 调用管理员创建API，发送 `status: 2` (禁用)
2. 验证返回结果成功
3. 查询创建的用户，验证状态是否为 `1` (正常)

**预期结果**: 状态总是为 `1` (正常)，忽略前端传入的 `status`

#### 测试2: 管理员更新时状态字段处理
**目标**: 验证管理员更新时发送的 `status` 字段是否被接受

**测试步骤**:
1. 创建一个管理员用户
2. 调用管理员更新API，发送 `status: 2` (禁用)
3. 验证返回结果成功
4. 查询用户，验证状态是否为 `2` (禁用)

**预期结果**: 状态成功更新为 `2` (禁用)

#### 测试3: 前端用户创建时状态字段处理
**目标**: 验证前端用户创建时发送的 `status` 字段是否被接受

**测试步骤**:

2. 验证返回结果成功
3. 查询创建的用户，验证状态是否为 `2` (禁用)

**预期结果**: 状态成功更新为 `2` (禁用)

### 2.2 密码策略一致性验证

#### 测试4: 管理员创建时默认密码
**目标**: 验证管理员创建时密码为空是否使用默认密码 "123456"

**测试步骤**:
1. 调用管理员创建API，不发送 `password` 字段
2. 验证返回结果成功
3. 使用默认密码 "123456" 登录，验证是否成功

**预期结果**: 可以使用 "123456" 登录

#### 测试5: 前端用户创建时默认密码
**目标**: 验证前端用户创建时密码为空是否使用默认密码 "123456"

**测试步骤**:
1. 调用前端用户创建API，不发送 `password` 字段
2. 验证返回结果成功
3. 使用默认密码 "123456" 登录，验证是否成功

**预期结果**: 可以使用 "123456" 登录

#### 测试6: 密码复杂度验证
**目标**: 验证两个模块的密码复杂度验证逻辑

**测试步骤**:
1. 开启安全开关 (`security.password`)
2. 尝试创建用户，密码为 "123456" → 应该成功（跳过验证）
3. 尝试创建用户，密码为 "password" → 应该失败（缺少数字）
4. 尝试创建用户，密码为 "password123" → 应该成功

**预期结果**: 
- "123456" 成功（跳过验证）
- "password" 失败
- "password123" 成功

### 2.3 超级管理员保护机制验证

#### 测试7: 删除最后一个超级管理员
**目标**: 验证删除最后一个超级管理员时的错误处理

**测试步骤**:

**预期结果**:

#### 测试8: 降级最后一个超级管理员
**目标**: 验证降级最后一个超级管理员时的错误处理

**测试步骤**:

2. 尝试修改该超级管理员的角色，移除超级管理员角色
3. 验证前端是否显示友好的错误提示

**预期结果**: 
- 后端抛出 `USER_CANNOT_DEMOTE_SUPER` 异常
- 前端显示 "系统中至少保留一个超级管理员，无法降级"

### 2.4 前端UI一致性验证

#### 测试9: 管理员创建页面状态选择
**目标**: 验证管理员创建页面的状态选择是否应该移除

**测试步骤**:

3. 尝试选择"禁用"状态并创建

**预期结果**: 
- 状态选择控件应该移除（因为后端不接受）
- 或者状态选择控件应该禁用

#### 测试10: 前端用户创建页面状态选择
**目标**: 验证前端用户创建页面是否应该添加状态选择

**测试步骤**:
1. 打开前端用户创建页面
2. 查看是否有状态选择控件
3. 验证后端是否接受状态字段

**预期结果**: 
- 如果后端接受状态字段，前端应该添加状态选择控件
- 如果不需要，前端可以不添加

## 四、风险评估

### 高风险
1. **状态管理逻辑不一致**: 可能导致用户状态混乱
2. **超级管理员保护机制失效**: 可能导致系统没有超级管理员

### 中风险
1. **密码策略不一致**: 可能导致用户登录问题
2. **前端UI不一致**: 可能导致用户体验问题

### 低风险
1. **文档不完整**: 不影响系统运行

- [ ] 前端用户创建时接受 `status` 字段

- [ ] 不能降级最后一个超级管理员

- [ ] 管理员创建页面的状态选择被移除或禁用
- [ ] 前端用户创建页面的状态选择符合需求
