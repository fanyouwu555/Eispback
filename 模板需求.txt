服务端需要提供给前端和客户端相关功能
服务端相关数据表定义:
	模板表
		主键id
		模板名称
		模板描述
		模板封面图路径
		资源路径
		详细描述
		创作者
		创作时间
		上线时间
		下载人数
		顶级分类id
		一级分类id
		二级分类id
		版本号
		更新时间
		难度等级
		有效时间
		是否付费
		费用类型
		下载次数
		状态 
			下架
			上架
			违规
			正常
		收藏数
		
		
	顶级分类表
		Id	Parent_id	Name	level
		1	0	常用模板	0
		2	0	比赛模板	0
		3	0	特殊类模板	0
		
	一级分类表
		Id	Parent_id	Name	level
		1	1	小车类	1
		2	2	机械臂类	1
		3	3	机器人类	1
		
	二级分类表
		Id	Parent_id	Name	level
		1	1	小车类巡线	2
		2	2	小车类陀螺仪	2
		
		
前端页面需求：
	模板分类页
		用于创建模板分类树
		1.模板分类数最多设定为3层
		2.一个顶级分类下可以添加多个一级子分类，一个一级子分类可以添加多个二级子分类
		3.每添加一个分类需要在表中添加一行对应的数据，id递增，数据包括 Id	Parent_id	Name	level，名称不能为空，新增后的分类节点只可以修改名称
		4.节点删除需要从末端节点开始删除，如果纯在子几点则不可以删除，删除完的子几点需要在数据表中标记废弃或直接删除。后续数据操作的时候需要排除标记废弃的数据行
	模板管理页 
		平台所有模板的核心管理页面。
		1.模板创建功能页（创建一个模板）
			a.新建模板
				模板名称，
				模板描述，
				详细描述，
				创作者，
				创作时间，
				模板分类树（对应三级分类的id）,
				模板封面图路径（通过前端页面上传到指定资源服务器阿里云OSS，并记录相对上传路径），
				模板资源路径（通过前端页面上传到指定资源服务器阿里云OSS，并记录相对上传路径），
				版本号，
				版本描述，
				上线时间，
				有效时间，
				难度等级，
				是否付费，
				费用类型
		
				
		
		2.模板展示列表：（列表可以通过id，模板名称，状态，进行筛选查看）
			模板ID、模板名称、所属分类（顶级名称-一级名称-二级名称）、当前版本、状态，操作（详情，编辑，版本，上下架，删除）
			
		3.模板版本编辑页：
			上传新封面、上传新的资源、版本号，版本描述，更新时间，上线时间，有效时间，难度等级，是否付费，费用类型。
		
		4.详情展示
			展示模板表中记录的所有数据信息

			
		
unity客户端需求
	需要从服务器获取如下数据接口api：
		1.3个分类表的实际数据库数据（排除了废弃的数据）
		2.返回所有有效的模板数据列表数组（有效模板是指上架的并且在有效时间内的）
			模板数据：
				主键id
				模板名称
				模板描述
				模板封面图路径
				资源路径
				详细描述
				创作者
				创作时间
				上线时间
				顶级分类id
				一级分类id
				二级分类id
				下载次数
				版本号
				更新时间
				难度等级
				有效时间
				是否付费
				费用类型
